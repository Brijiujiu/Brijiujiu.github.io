Pycharm在Linux环境下run报错解决方案

适用Manjaro的AUR源安装的Pycharm v2020.1

Linux的开发环境配置确实方便，但是出这码事是我没想到的，妈的。老子折腾了一整天。

和 Windows 下的 Pycharm 有点不一样，在 windows 下可以正常工作的目录结构在linux下直接瞎眼。

环境：

系统： Manjaro Linux X86-64
内核: 5.7.9-1-MANJARO
Python环境： Anaconda3、system python

pycharm工作目录结构：

/home/usrname/
|
PyObject —————— Fibonaci.py 、DeltaCompute.py、and so on
|
DiskGuard

在这种情况下，选择Pycharm打开PyObject目录
然后直接编辑Interpreter，启动Script Fibonaci.py，这是一断有导入turtle模块用作斐波那契函数螺旋绘图的程序
Run！
就报错了。

ImportError: cannot import name 'log' from partially initialized module 'math' (most likely due to a circular import)

尝试了更多文件，确定报错信息主要集中在导入的模块所引用的math或者Random，但是anaconda环境不可能出现缺少模块的错误。

因为报错信息实际上和错误原因没有很大关联，所以仅放一段代表作为记录

而如果没有导入任何模块或者直接在 Python Console 下执行，则不会出错

错误原因

在尝试更换工作目录、检查目录权限之后，我发现在同一个目录下只有一个py文件时执行run并不会出错

可能是Pycharm单一目录一个Object的机制，在同一个目录放置多个无关联的Python文件可能产生判断失误之类的

无常

看了三两遍「朝花夕拾」，十分有趣，当中不时夹着些讽刺“正人君子”的话语，当中「活无常」一章我反复地看，其一是对绍兴民俗的新奇，还有便是为了那闻所未闻的「目连戏」和「目连嗐头」

目连在我的印象里是那个下地狱救母的仙儿，民间众说纷纭，是仙是佛还是人具体记不得了，但隐约觉得「宝莲灯」的主角似也叫目连，便一下对上了，而这绍兴民间的目连戏和目连有甚关系，我是不大晓得的。

活无常这个角色，其他的艺术作品中我是见过多次的，一般唤作「白无常」，而他的吊死鬼兄弟死有命，民间多叫「黑无常」，而「死有命」这个“本名”却是我头一回见到，但一想也觉得合理。至于活无常出场需吹响的头，才是我所关注的，它声音如同“Natu Natu Natututu”，只看字母是无法体会它怎么叫唤，想必是很有趣的罢；网络上特地找过，却没找到关于这种「加长号筒」的录音，于是我的好奇心便起来了。

直到今天，我仍然不知道目连嗐头怎么叫，但不时想起它我的眼前就飘过一行“Natu Natu Natututu”，书里描写的事大多久远，唯有这神性而欢快的声音，仿佛跨过百年时光，一直响在我的近旁。

trojan代理从搭建到应用

事先声明：本人对trojan及此类型软件的拓展使用方法一概不知，爱国爱党。

请勿作非法用途

众所周知，一般来说，基于解决为适应我国国情而做出的部分国际互联网限制所导致的学术信息交流不方便等问题而提出的大多数加密流量型代理协议(SS/SSR等)已经不再稳定和安全，众多网友迫切地寻找一款真正能够满足日常需求的代理软件。于是，v2ray以及Trojan就应运而生。二者都是伪装流量类型的代理方式。
Trojan in github

  • v2ray是完整的网络框架，对服务器及用户的资源占用是比较大的。但是使用比较广泛。
  • trojan可以说是v2ray的简化版，由于使用C++编写以及简化了一层加密，理论速度会比v2ray高，负载也较小。但是Trojan不能套接CDN加速。

本文主要讲述trojan在Azure服务器的安装过程。

一、搞到一台VPS

文中采用Azure的日本东部免费订阅，Ubuntu服务器作为示例。

服务器必须开放80及443端口

方法自查。

二、DNS解析配置

推荐Cloudflare，不要使用VPS自带的dns

在cloudflare填写配置完成后需要几个小时才能生效，判定方法为ping出的ip是否为服务器ip。

三、开始配置服务端

  1. 使用SSH连接服务器，以root用户登录

  2. 注册SSL证书，如果已经有证书可以跳过这步。

    1. apt update&&apt upgrade&&apt install certbot
    2. certbot certonly --standalone -d [你的域名]
    3. 按照提示填写信息即可。

  3. 完成生成SSL证书后，可运行certbot certificates 来获取证书路径，分别有两个pem文件。

  4. bash -c "$(curl -fsSL https://raw.githubusercontent.com/trojan-gfw/trojan-quickstart/master/trojan-quickstart.sh)"

  5. apt install nginx 此为安装nginx框架搭建伪装网页

  6. 配置Trojan： vim /usr/local/etc/trojan/config.json

    按i编辑"password":中的两个密码，默认使用第一个。接着修改cert和key,分别填入ssl证书（pem文件）的绝对路径。

    完成后ESC后：wq保存退出。

  1. 就此安装完成，接下来配置开机自启及启动

以上服务端配置完成，可ss -lp | grep trojan检测运行状况，有输出即正常，无输出检查步骤。

四、客户端配置

windows可使用trojan.exe或clash、V2rayN，linux参考windows。Mac未知可自行查询。
路由器多数第三方固件可使用三月中旬后更新的clash。

clash配置参考（第102、104、106、128、130、132修改为自己的参数，其它依用户水平自己调整。）：
https://raw.githubusercontent.com/Dabrit/LinkFiles/master/clash.yaml

三体:生存与人性间的抉择

给岁月以文明，而不是给文明以岁月 —— 《三体：黑暗森林》刘慈欣

第一次看到这句话，是在三体中人类文明大低谷之后的纪念碑雕塑上，原文是帕斯卡（Blaise Pascal 1623—1662）的“给时光以生命，而不是给生命以时光”(To the time to life, rather than to life in time)， 大刘稍微加以改动，使它更加符合《三体》系列的内涵。
文明的历程其实和生命个体相似，而给时光以生命亦或者相反，代表的是两种生活的状态。给岁月以文明是三体中人类文明选择的存在方式，也是其在三体式的黑暗宇宙中无法继续生存的原因。

技术概念

三体的脑洞是我读过的硬科幻小说中最牛逼的，它牛逼就牛逼在当中涉及到的科技概念涉及范围极广，通信工程有红岸，材料学有飞刃，生物技术有基因针对性病毒、冬眠和思想钢印，计算机科学有三体人的人列计算、智子和虚拟现实技术的概念，还有一些在当今还算前沿领域的基于弦论或M理论的维度科学和核聚变等等，整本书可以说是技术爱好者的盛宴，据说大刘是在误诊绝症的情况下把自己多年的存货全写出来了，果然是绝症出人才。

社会描写

科技只是三体这一神作的冰山一角，作为一部描写时间跨越18904922年的故事的作品，它展现了两个文明在四个世纪中的爱恨情仇，慢慢揭开三体宇宙中的黑暗真相，无论是三体人在烈日与严寒中数百次重生的挣扎求生还是人类文明面对危机的各种表现，都给人以震撼而真实无比的感受。

我的解读

地球往事

《地球往事》的主要目的是铺垫背景。叶文洁因为悲惨经历决定背叛自己的文明，向遥远的星空回复，给艰难过活又即将死透的三体文明带来一线生机，重新回想起来，第一部已经展现出了一种无声的技术碾压，导致我在看到“闪烁的宇宙”的时候产生了一种神圣的感觉，在巨大的技术鸿沟面前人类和三体的差距就如同虫子和人类一样，在本书末章三人在麦田里看到蝗灾，放言“虫子从未被消灭”，为人类日后长达几个世纪的抗争拉开了序幕。

黑暗森林

本书开头是叶文洁和罗辑在杨冬墓前的谈话，叶文洁离开杨冬墓前往ETO的聚会，吹爆蚂蚁视角的描写。谈话内容标志着宇宙社会学的诞生，是日后黑暗森林理论发现和逆风翻盘的重要基础。罗辑其实是一个很普通的人，如果没有这次谈话他会在大学中教书混过一辈子，但他因为ETO屡次暗杀被联合国强行赋予了使命，成为了四位面壁者之一。另外关于脑中造人的文段为我打开了新世界的大门。在其余三位面壁者中除了泰勒的量子舰队计划，其它两人虽然半路流产，但都为后世铺了路，曼努尔·雷迪亚兹的恒星氢弹为日后的油膜计划和太阳闪烁威慑创造了条件，在水星上的弹坑还为曲率引擎的暗中研究提供了场所。比尔·希恩斯的思想钢印使大量人类转变为失败主义者，后来有一艘战舰的主动“叛逃”应该也受过影响。说来讽刺，也为人类文明的胜利做了铺垫，在这种技术碾压下，只要活着就是胜利。

另外一个重要人物就是章北海，他可以算作是隐形的面壁者了，他曾经对无工质推进做出不可磨灭的贡献（杀光了支持工质推进的大佬），策划了自然选择号的逃离。可以说没有章北海就没有星舰地球，就没有银河系人类，黑暗战役后能进行恒星际航行的人类战舰就会一艘不保。

在《黑暗森林》中有少量的博弈论内容，令人感叹的是面壁者精妙绝伦的计谋，要设计出如此紧密的情节大刘必须对各种技术了解透彻应用灵活，各种反转也是看得拍案叫绝。

大低谷是这部作品中一个关键的转折点，正是在大低谷过后人类确立了自己的主流价值观“给文明以岁月而不是给岁月以文明”，即文明之所以存在是因为有文明而不是因为存在。大低谷之后人类将尊严和人性放在了生存之上，成为了意义之塔的顶端，而这也可以侧面解释为什么太阳系人类文明中的其它人无法得出黑暗森林理论，因为他们认为宇宙或许不美好，但文明一定要是美好的，必须将价值实现放在生存的前面。在这方面三体是截然相反，两者的生存环境不同导致了不一样的思想主流，间接影响了各自在大宇宙中的最后结果。

死神永生

杨冬的想法“这个宇宙被生命改变了多少？”是我之前从来没有想过的，也是第三部要揭示的大背景的导致原因。

故事从云天明身上说起，他的脑子被人剜出来装飞船里发射给三体人当间谍了，具体是哪边的间谍还不一定，三体人咋突然学会了欺骗了呢？第二部中提出人脑工作涉及量子论，智子应该无法直接观察研究，给他们送颗脑子就不一定了。

关于程心，这确确实实是个圣母，但她是符合人类的整体价值认同的。大低谷之后人类决定给文明以岁月，决定相信真善美，在了解了宇宙的黑暗真相后被战后蜜月迷惑了一下，又开始怀疑起来。程心的上位是文明的选择，程心不按下威慑按钮是文明的选择，程心一次次毁灭掉人类文明生存的希望是文明的选择，太阳系人类文明最终消失在黑暗冷酷的宇宙中也是文明的选择。程心一次次在生存与人性之间做着选择，最终以一颗圣母的心，一颗真挚美好、“如同露珠一般”的心，将太阳系人类送离了黑暗的现实。
太阳系人类从大低谷之后就已做好了消亡的准备。

关于降维打击的部分还是很震撼的，整个过程就像史诗一样，也是为相信着光明的太阳系人类文明的送行。

正文结束

部分显示器A卡输出开机过程不显示主板商标（自检过程）的解决方法

教程经验由笔者实践得出，仅作备忘记录。

问题主要体现在主机按下开机键到windows系统登陆界面亮起的阶段，正常情况应该显示主板厂商的商标之后win10窗口转圈圈，但是这个问题会使登录界面亮起前的开机过程中屏幕完全黑屏，即没有信号输入显示器，最大的问题就是无法进入BIOS和无法选择多系统启动项
笔者的问题只在HDMI连接显示器的情况下出现，如果显卡输出HDMI转接VGA再接入显示器则显示正常，但是不能使用freesync。

参考硬件信息：
  • CPU AMD RYZEN7 2700
  • GPU AMD 蓝宝石RX580 8G超白金
  • 显示器 三星S24R35X

笔者查看过网络上大部分关于这类问题的资料，大部分问题都出在三星显示器和AMD显卡的搭配组合上。我怎么就这么倒霉

这种情况需要满足两种条件才可以解决：

  • 1.你的显卡支持UEFI启动，大部分近年的显卡都支持。
  • 2.你的主板需要支持并启用纯UEFI启动，关于这句话的涵义后文会解释
  • 3.请确认磁盘分区表类型为GPT，以便于使用纯UEFI启动。
  • 如果使用Windows系统，请确认系统以UEFI引导方式安装，以防止更改为纯UEFI引导后发生windows引导错误的情况。如果出现此情况请自行解决。

于是解决问题就可以分为几个步骤：

1.打开GPU-Z查看显卡是否支持UEFI。不支持的话后面别看了


(图片来自网络，注意BIOS VERSION那行最后的UEFI，打勾的支持UEFI)

2.打开BIOS找到windows启动管理（可能叫别的名，有的厂商开启WHQL驱动支持就可以)

里面有启动选项，如果是CSM就将其改为UEFI启动。可能会遇到兼容性问题，但是一般还可以加快启动速度
完成以上步骤之后重启就完事。不出意外已经可以正常显示自检界面。如果出了意外，接着看。

极端情况

完成以上步骤后，启动过程中如果长时间黑屏后提示There is no GOP support detected in this card或者类似的文字并尝试让你重新使用CSM启动。

在保证显卡支持UEFI的情况下就只可能是显卡的VGA BIOS版本太低，与显示器不兼容。
这种情况就需要重新刷显卡的BIOS

以下为教程。

  • 1.打开GPUZ。在BIOS version后面有一个分享图标，点击并把显卡的bios备份一份，如果对自己有信心可以省略这一步。
  • 2.下载ATIFLASH并打开。可能仅适用AMD显卡
  • 3.找到并下载你的显卡BIOS最新版本，后缀是rom，可以在https://www.techpowerup.com/vgabios/查找。
  • 4.打开AITFLASH，点击载入图像
  • 4.选择你下载的rom文件，再点程序就可以刷入。

一般完成后重启用HDMI连接就可以正常显示自检，如果你不幸翻车，可以刷回备份的BIOS。

狼と香辛料观感

202003051559二记

【狼と香辛料】动漫 一二季观后感

  • 非常打动人的作品，后劲足，念念不忘。但是又不好说出是哪里好，哪里打动人心。

人物

男主罗伦斯是一个在中世纪平常可见的旅行商人，最大的梦想就是开一家自己的店，善良，精明，情商高。
这里不得不牵扯到作品的事件背景，大概在中世纪末即14-16世纪的时间段，在现实中对应的时代重大事件其中之一是资产阶级兴起，作为商人的罗伦斯可谓占据了时代领先的地位。
在遇到女主之前，罗伦斯应该有一个非常鲜明的特征，那就是孤独；旅行商人的事业就是每天驾着马车在各地周游，在麦子地里捡到女主之前想必是寂寞得长毛了。

女主赫萝是我到现在为止接触过的ACG作品中最完美的女人，狼的坚忍和狐的狡黠融为一体，并且人格上独立。她害怕孤独。因为遵守约定成为掌管丰收的狼神，她远离家乡独自一人守护麦田几百年，会认为孤独是足以致死的病症也就可以理解。赫萝的特殊种族在狼辛的世界观下还是比较常见的，在故事的进程中就有遇到另外两个“异教神明”也就是动物成精，与短寿种人类不同，他们通常拥有相对人类来说极长的寿命，是故事后段两人心理斗争的缘由之一。
我想到一个生物学问题，如果这个种族的繁衍效率能再提升一点（在与人类通婚的方面）是不是能让后世人类种族的平均寿命大幅度增加？兽耳满地跑岂不更是美哉。
另外，她的语言很有特点，如果有认真听或者关注日文原著可以发现赫萝使用的近似在日本被称为 “廓言葉”（或称「花魁言葉」）的特殊语法结构和说话方式，一般是游女以及落魄贵族女孩所使用，在欧洲也有出现类似语言地位的事物（待笔者考证），现代也有人使用或专门研习。

故事

写的是两个旅人的相遇，男主继续行商，女主寻找故乡，在去北方的路上，其实互相都动了心思，但都认为在一起的结果会让对方的梦想破灭，所以一直憋着。看的时候一直以为女主怕的是寿命的不平衡，是男主百年之后自己独自面对漫长时光的迷茫与痛苦，但正如原作所说，“旅行唯有出发前最愉快,狗儿唯有叫声最吓人,女人唯有背影最美丽”，赫萝害怕的更是互相之间过于熟悉之后感情的逐渐淡去甚至是背叛，所以才会提出“不如就此分开”。

罗伦斯在第二季的最后明白了所谓的“期待”，我的理解大概便是追求事物本身的过程，而不必过于在意结果。
“无论如何都想一直注视着，这个过程才是最快乐的时光吧”

这是一个结局注定悲伤而一时又看不出悲伤的故事。

制作和音乐

从动漫质量本身说起，作画处于两个日本动漫美术风格的过渡阶段，配乐很有中世纪的浓厚风格，再加上作品的背景本身就是中世纪德国，非常有感觉。
第一季OP（旅の途中）和ED（リンゴ日和），一个道尽旅途的孤独、相遇的幸运，另一个倒像是角色歌，日式英文萌的发慌，就是听多了影响语感。

另外，狼辛一二分别在08、09年播出，而第三季直到现在（2020年3月）依然遥遥无期。不知道为什么最近开始对老番感兴趣，也许是新番大都过于浮躁，其中将女性角色制成迎合不同死宅口味的没有灵魂的人偶的作品不在少数，而平淡又令人回味的作品寥寥无几。外加幻想中世纪这种少见的题材远离现实世界，人物设定和叙事方式给人以很特别的感觉，就像是一个真实的童话一样。想到过去的十一年时光，我错过了多少像这样的好东西，感觉有点惋惜和兴奋，也忍不住想到这部作品制作、发行、风靡的时候，我在什么地方、做着什么事。

十一年的时光里，足够发生很多的事，狼辛的热度已经降下来了，但从今天的人气依然可以看出它当初是如何的一部霸权,而制作第三季应该属于有生之年的范畴了。不过倒是可以期待一下狼崽子的故事。

杂谈

狼辛的货币体制很有意思，在历史上的欧洲中世纪末期的确出现大量通行的货币种类，因为多个国家多个朝代的货币都在继续使用，于是出现专门兑换货币的货币商人职业，在狼辛中也有提到。这里可以看出支仓冻砂对中世纪历史的了解的确非常深刻。

狼辛的感情线和商战描写都很饱满，值得一看。
同时这部动漫给我的书单添加了一些新东西，

  • 1.《狼与香辛料》、《狼与羊皮纸》

    我对中世纪的文明状态有挺高的兴趣，狼辛的世界是介于中世纪和文艺复兴的时期，神权衰弱而人文兴起，番剧只制作到小说的三分之一不到，还是想了解十七个章节中的全部内容。羊皮纸算是后续，讲的是男女主的女儿被徒弟拐跑的故事。话说历史总是惊人相似。
  • 2.《叔本华思想随笔》
    支仓冻砂在小说十六章序中说明过，叔本华为他的创作中人物情感的变化提供了很大的灵感。那个出生在十八世纪的哲人，曾经对他卖畅销书的母亲说过:你的书将在几十年内被世人冷淡，而我的书将成为后世的灵感源泉。
    事实证明是这样的，书架上的《叔本华思想随笔》已经落灰许久，但我初看的时候确实感受到了共鸣，现在是时候重新捧起。
  • 3.《The Golden Bough（金枝）》-弗雷泽
    人类文明研究、神秘学和宗教巫术研究的经典，据了解赫萝原型来自于欧洲类似“五谷狼”的信仰，《金枝》的四十八章中有比较详细的描述。

暂时到这里，我要补小说了

移动硬盘陌生设备读取识别记录及预警

此文章原提交为32届科技创新技术论文，现发表于博客

移动硬盘陌生设备读取行为自动记录及预警软件

该项目提供了一种移动数据硬盘丢失找回的方法，可以达到移动数据储存设备（如U盘等）对丢失后被接入未经允许的Windows设备并进行读取的行为作出记录和预警，以方便失主取得关于丢失硬盘接入设备的相关信息，便于找到移动硬盘。
该项目从某些即时通讯软件的异地登陆提醒功能得到启发，结合我自己曾多次丢失移动硬盘的经历，经过反复构思和设计，逐步形成预警软件雏形。

  • 软件工作过程简述：
    当硬盘接入设备时，自动启动硬盘主控芯片内预编译的固件（firmware），运行已编译完成的程序主体，识别计算机信息，对当前用户身份进行识别，并反馈预警信息至主人用户指定邮箱，以便使主人用户及时知晓硬盘的被使用地点、时间、计算机名甚至接入的计算机桌面情况，便于及时找回丢失硬盘。

  • 项目开发过程：
    最初考虑过完全使用C语言开发，后因python语言的高效、可移植性和可拓展性而选择python语言作为主要开发语言，采用C语言编译主控芯片固件。
    程序用于储存数据的方式最初为JSON半结构化数据，后考虑到用户本地环境的可读性，转为使用最为广泛的txt文本格式。
    也曾构思过在程序未成熟的阶段搭建服务器作为信息转发至用户邮箱的中介，后决定采用python的email及smtplib模块实现功能。
    原本计划采用autorun.inf文件实现在windows系统上的自动运行功能，又因为autorun.inf脚本的易误杀性和不稳定性而使用用硬件辅助实现自动运行功能,在硬件方面先后考虑了attiny85开发板和Leonard开发板，后因免驱动功能实现而选择Leonard开发板。
    又在识别计算机身份的基础上添加了识别当前用户身份，添加人脸识别以辨别读取操作合法性。

  • 软件工作过程：
    当硬盘接入设备时，自动启动硬盘主控芯片内预编译的固件（firmware），与服务器同步软件主体，运行已编译完成的程序主体，识别计算机信息，对当前用户身份进行识别，
    随后反馈预警信息至移动硬盘失主指定邮箱，以便使主人用户及时知晓硬盘的被使用地点、时间、计算机名及非法读取者的面部照片，便于及时找回丢失硬盘与辨认非法读取者。

  • 软件工作原理：
    移动硬盘接入未知设备后，硬盘主控芯片内使用C语言编译的固件将会自动锁定文件路径，通过模拟HID的方式调用windows系统的运行指令，开启开启powershell同步服务器中的软件主体，执行指定路径中python程序编译的exe可执行文件，在本地自动运行并完成身份识别及预警和记录。
    通过time、socket、platform模块收集时间、设备系统版本等信息，urllib2模块爬虫收集设备公网IP地址等信息。
    调用opencv、base64等模块进行摄像头的操作、图片的拍摄与编码。
    调用人脸识别api应用程序接口，实现执行当前读取操作用户的身份与硬盘拥有者身份的对比。
    使用email、smtplib等模块实现邮件预警的发送功能。
